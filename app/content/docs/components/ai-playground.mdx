## Features

The AI Playground component includes:

- **Multiple AI Models**: Support for GPT-4, Claude, Llama, and more
- **Real-time Chat Interface**: Interactive conversation with AI models
- **Configuration Controls**: Adjust temperature, max tokens, and system prompts
- **Message Management**: Copy messages, clear chat history
- **Loading States**: Visual feedback during API calls
- **Error Handling**: Graceful error display and recovery
- **Responsive Design**: Works on desktop and mobile devices

## API Reference

### AIPlayground

The main playground component.

| Prop | Type | Default | Description |
| --- | --- | --- | --- |
| `className` | `string` | - | Additional CSS classes to apply |
| `defaultModel` | `AIModel` | `"gpt-4"` | Default AI model to use |
| `defaultSystemPrompt` | `string` | `"You are a helpful AI assistant."` | Default system prompt |
| `onSendMessage` | `(message: string, config: PlaygroundConfig) => Promise<string>` | - | Custom message handler |
| `showSettings` | `boolean` | `true` | Whether to show the settings panel |

### PlaygroundConfig

Configuration object passed to the message handler.

```tsx
interface PlaygroundConfig {
  model: AIModel
  temperature: number
  maxTokens: number
  systemPrompt: string
}
```

### Message

Message object structure.

```tsx
interface Message {
  id: string
  role: "user" | "assistant" | "system"
  content: string
  timestamp: Date
  model?: AIModel
}
```

## Integration Examples

### With Next.js API Routes

```tsx
// app/api/ai/route.ts
import { NextResponse } from "next/server"

export async function POST(request: Request) {
  const { message, model, temperature, maxTokens, systemPrompt } = await request.json()

  // Call your AI service here
  const response = await callAIService({
    message,
    model,
    temperature,
    maxTokens,
    systemPrompt,
  })

  return NextResponse.json({ response })
}
```

### With Authentication

```tsx
export default function AuthenticatedPlayground() {
  const { user } = useAuth()

  const handleSendMessage = async (message: string, config: PlaygroundConfig) => {
    if (!user) {
      throw new Error("Please sign in to use the AI Playground")
    }

    // Include auth token in request
    const response = await fetch("/api/ai", {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Authorization": `Bearer ${user.token}`,
      },
      body: JSON.stringify({ message, ...config }),
    })

    const data = await response.json()
    return data.response
  }

  return <AIPlayground onSendMessage={handleSendMessage} />
}
```

### With Rate Limiting

```tsx
export default function RateLimitedPlayground() {
  const [requestCount, setRequestCount] = React.useState(0)
  const MAX_REQUESTS = 10

  const handleSendMessage = async (message: string, config: PlaygroundConfig) => {
    if (requestCount >= MAX_REQUESTS) {
      throw new Error("Rate limit exceeded. Please try again later.")
    }

    setRequestCount(prev => prev + 1)

    // Your API call here
    const response = await callAPI(message, config)

    return response
  }

  return (
    <div>
      <AIPlayground onSendMessage={handleSendMessage} />
      <p className="text-sm text-muted-foreground mt-2">
        Requests remaining: {MAX_REQUESTS - requestCount}
      </p>
    </div>
  )
}
```

## Troubleshooting

### Common Issues

1. **Messages not sending**: Ensure your `onSendMessage` handler is properly configured and returns a string
2. **Styling issues**: Check that all required shadcn/ui components are installed
3. **Model not available**: Verify the model is included in the `AI_MODELS` array
4. **Copy to clipboard not working**: Requires HTTPS or localhost

### Debug Mode

Enable debug logging by setting the `DEBUG` environment variable:

```tsx
if (process.env.NODE_ENV === "development") {
  console.log("AI Playground Config:", config)
  console.log("Messages:", messages)
}
```
